## 1. 서비스 개요

국내 상장 주식 시장을 산업(섹터) 단위로 상시 모니터링하여, **"지금 어떤 산업의 어떤 기업을, 왜 봐야 하는지"**를 자동 분석해 매일 아침 대시보드와 텔레그램으로 제시하는 1인용 재무 분석 서비스. 거시 → 산업 → 기업으로 좁혀가는 탑다운 투자 로직을 데이터 파이프라인과 시그널 엔진으로 시스템화한다.

**운영 전제: 사용자 1인, 비용 최소화 최우선.** 서버를 두지 않는 정적(static) 아키텍처를 채택한다. LLM은 Gemini 무료 티어를 기본으로 사용하여 고정비/변동비를 0원에 수렴시키되, 필요 시 일부만 유료 모델로 보완한다.

### 1.1 투자 로직 → 기능 매핑

| 단계 | 투자 로직 | 서비스 기능 |
|---|---|---|
| 1) 거시 | 거시경제 정보 취합 및 국면 판단 | 금리·환율·수출입·물가 자동 수집, 추세 표시 |
| 2) 산업 선별 | 업턴 전환 등 긍정 시그널로 분석 대상 산업 선정 | **목표주가 컨센서스 모멘텀** 중심 섹터 스코어링 |
| 3) 산업 분석 | 주요 애널리스트 리포트 참고 | 산업별 Top 3 애널리스트 워치 + 리포트 이벤트 DB + 요약 |
| 4) 기업·ETF | 대표 기업·ETF의 글로벌 경쟁력, 이익률 추이, 주가 모니터링 | 산업별 대표 종목·ETF 자동 매핑, 재무·수급·주가 추적 |

## 2. 핵심 시그널: 섹터 목표주가 모멘텀

1순위 시그널은 **"섹터 내 애널리스트 목표주가가 동시에, 큰 폭으로 상향되는 현상"**. 개별 상향은 종목 이벤트일 수 있으나 섹터 단위 동시 상향은 산업 펀더멘털 변화의 증거로 본다. 절대 수준이 아닌 **변화(revision)**를 추적한다.

### 2.1 파생 지표 3종 (윈도우 4~8주)

| 지표 | 정의 | 해석 |
|---|---|---|
| 상향 폭 (Magnitude) | 섹터 내 목표주가 변경률 평균 | 얼마나 크게 올리는가 |
| 상향 확산도 (Breadth) | (상향 − 하향) / 전체 리포트 수 | "다같이 오르는가" — 쏠림 배제 |
| 상승 여력 (Upside Gap) | 섹터 평균 (목표가 − 현재가) / 현재가 | 진입 매력 잔존 여부 |

우선 후보: Breadth ≥ +0.6 & Magnitude 상위. 대시보드 메인 = Breadth × Magnitude 산점도(버블 크기 = Upside Gap), 우상단 큰 버블이 "지금 주목할 산업".

### 2.2 보정 장치 (필수)

1. **후행성 보정**: 주가 상승률 대비 목표가 상향률 비교 — 목표가가 주가를 앞서면 선행 시그널, 뒤따르면 추격 조정으로 감점
2. **희소 섹터 보정**: 윈도우 내 최소 5건 미달 섹터는 시그널 보류 + "데이터 부족" 표시 (산업별 커버리지 밀도 등급 연동)

### 2.3 보조 시그널 (2차 필터)

- 산업: 업종지수 골든크로스, 업종 ETF 거래대금 증가, 수출 전년비 플러스 전환, 외국인 업종 순매수 전환
- 기업: 영업이익률 2분기 연속 개선, 컨센서스 상향, 외국인 지분율 증가, 글로벌 점유율(리포트 인용 기반 LLM 보조)

## 3. 산업 분류 (부속 문서 1 요약)

- Level 1 = WICS 10개 대분류 (네이버·다음·에프앤가이드가 쓰는 실무 표준 → 데이터 결합 용이)
- Level 2 = 31개 산업, 애널리스트 커버리지 단위로 재구성 (조선·방산·전력기기·2차전지 등). MECE: 모든 종목은 매출 최대 사업 기준 단일 귀속, 잔여는 "기타" 버킷
- **귀속의 1차 기준은 리포트의 업종 표기가 아니라 종목코드** — 증권사별 업종명은 비표준이므로 alias 테이블은 산업 리포트에만 보조 사용
- 2차전지는 WICS와 의도적으로 다른 통합 정의 → 종목 오버라이드 목록으로 관리

## 4. 애널리스트 트래킹 (부속 문서 2 요약)

- 산업별 Top 3 워치리스트: 어워드 시드(매경·한경비즈니스·에프앤가이드 산업별 수상자)로 첫날부터 가동 → 6개월 후 자체 적중률 엔진(목표가 괴리율 + 방향 적중 + 표본 가중)으로 분기 자동 교체
- 리포트 수집 3-Tier: 출시 감지·알림은 전체 리포트 100% 보장(메타데이터 상시 공개), 원문 공개분만 LLM 요약, 비공개분은 메타데이터 카드(제목·목표가 변경·링크)

## 5. 최소비용 아키텍처 (v2.0 핵심 변경)

### 5.1 설계 원칙: 제로-서버

상시 구동 서버·관리형 DB·클라우드 VM을 전부 제거한다. 모든 연산은 하루 1~2회의 배치로 충분하므로(시그널은 일 단위), 스케줄 실행 + 정적 산출물 구조가 기능 손실 없이 비용을 0원에 수렴시킨다.

```
┌─ GitHub Actions (cron, 무료) ── 매일 장 마감 후 + 오전 1회 ─────────┐
│  1) 수집: 네이버 리서치 크롤러 / pykrx 주가·수급 / DART / ECOS      │
│  2) 적재: SQLite 단일 파일 (repo 커밋)                              │
│  3) 계산: 3대 모멘텀 지표, 보정, 스코어링, 워치리스트 매칭          │
│  4) 요약: Top3 공개 PDF만 LLM 요약 (Gemini 무료 티어 기본)          │
│  5) 출력: 정적 HTML/JSON 대시보드 생성 → GitHub Pages 배포          │
│  6) 알림: 텔레그램 봇 푸시                                          │
└─────────────────────────────────────────────────────────────────────┘
```

| 구성 | 선택 | 월 비용 | 비고 |
|---|---|---|---|
| 스케줄러·연산 | GitHub Actions | 0원 | private repo 무료 2,000분/월. 일 배치 ~15분 × 30일 = 450분으로 여유 |
| 저장소 | SQLite 파일 (repo 커밋) | 0원 | 1인 규모(일 수십~수백 행)에 충분. git 이력 = 자동 백업 |
| 대시보드 | **정적 HTML + GitHub Pages** | 0원 | 서버 없음 → 잠들지 않고 항상 즉시 로드. 차트는 Plotly.js로 사전 생성한 JSON 렌더 |
| 알림 | 텔레그램 봇 | 0원 | |
| 거시·시세 데이터 | ECOS·KOSIS·DART·pykrx | 0원 | 전부 공식 무료 |
| LLM 요약 | Gemini API (Flash, 무료 티어) | 0원 (기본) | 아래 5.3 참조. 품질 이슈 시 Claude로 부분 교체 |

Streamlit Cloud(무료) 대비 정적 HTML을 택한 이유: 무료 앱은 미접속 시 잠들어 아침 확인 시 콜드스타트가 느리고, Python 런타임 의존이 생긴다. 정적 페이지는 항상 켜져 있고 모바일에서도 즉시 열린다. 단점(인터랙티브 필터 제약)은 사전 생성 JSON + 클라이언트 JS로 대부분 흡수 가능.

### 5.2 보안·접근

- repo는 private, GitHub Pages는 private 배포 불가 → 대안 2개 중 택1:
  (a) 대시보드 repo만 public으로 분리하되 종목 코드만 노출되는 수준으로 민감정보 없음 확인 후 사용 (간단)
  (b) Cloudflare Pages 무료 + Access(이메일 인증, 무료 50인까지)로 비공개 배포 (권장)
- API 키(Gemini, 텔레그램, DART)는 GitHub Secrets 관리

### 5.3 LLM 선택 및 비용 전략

**기본값: Gemini API (Flash, 무료 티어).** 이 프로젝트의 LLM 사용량(일 0~10건의 정형 요약 + 주 1회 브리핑)은 Gemini 무료 티어 한도 내에 충분히 들어가며, 비용 최소화 원칙에 가장 부합한다.

1. **선별 요약**: 전체 리포트가 아닌 *산업별 Top 3 애널리스트의 공개 PDF*만 자동 요약 (일 0~10건 수준)
2. **LLM 클라이언트 인터페이스 분리**: `summarize(text) -> dict` 형태로 LLM 호출부를 추상화하여 Gemini/Claude 등 모델 교체 비용을 최소화. 프롬프트와 출력 스키마(야마 1줄·근거 3줄·목표가 변경 맥락)는 모델에 독립적으로 설계
3. **온디맨드**: 워치리스트 외 리포트는 클릭 시에만 요약 (사실상 0건에 수렴)
4. **품질 모니터링**: 특히 목표주가·종목명·증권사명 등 수치/고유명사 추출 정확도를 운영 초기에 표본 점검. 시그널 신뢰성에 직결되는 항목이므로 오류율이 높으면 해당 항목만 Claude(Haiku, Batch API 50% 할인)로 교체
5. **예상 비용**: 기본은 0원. 일부를 Claude로 교체하더라도 일 5건 내외 기준 월 수천 원 이내(천 원 미만도 가능)

운영 데이터가 쌓이기 전까지는 추정에 의존하지 않고, Phase 3에서 실제 사용량·품질을 측정한 뒤 모델 구성을 확정한다.

### 5.4 확장 경로 (필요해질 때만)

데이터가 커지거나 실시간성이 필요해지면: SQLite → Turso/PostgreSQL(무료 티어), 정적 대시보드 → Streamlit/Next.js, Actions → 경량 VPS. 단, 1인 일 단위 모니터링 용도에서는 현 구조로 수년간 충분할 것으로 판단.

## 8. 단계별 로드맵

| 단계 | 기간 | 범위 | 완료 기준 |
|---|---|---|---|
| **Phase 1 — 데이터 심장** | 2~3주 | 네이버 리서치 크롤러(이벤트 적재) / 종목-산업 매핑(분류 v1.1 구현) / 주가 수집 / SQLite 스키마 / Actions 일 배치 / 애널리스트 시드 입력 | 무인 일일 수집 가동, 3대 지표 수동 쿼리 가능, 워치리스트 텔레그램 알림 작동 |
| **Phase 2 — 시그널 + 대시보드** | 2~3주 | 지표 자동 계산 + 보정 / 정적 대시보드(산점도·피드·드릴다운) + Pages 배포 | 아침에 휴대폰으로 주목 섹터와 근거 확인 가능 |
| **Phase 3 — 지능화** | 2~3주 | 거시·수급·재무 보조 필터 / LLM 요약·주간 브리핑(Gemini 기본) / 알림 고도화 | 브리핑 자동 생성·발송 |
| **Phase 4 — 검증·자동화** | 상시 | 시그널 백테스트(θ·가중치 보정) / 자체 적중률 엔진 → Top 3 자동 교체 / 글로벌 피어 비교 | 임계값·워치리스트가 데이터 기반으로 갱신 |
