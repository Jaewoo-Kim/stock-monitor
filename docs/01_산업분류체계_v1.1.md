## 1. 분류 설계 원칙

| 원칙 | 내용 |
|---|---|
| **MECE 보장** | 모든 상장기업은 정확히 1개의 Level 2 산업에 귀속 (중복 없음, 누락 없음) |
| **단일 귀속 규칙** | 복수 사업 영위 기업은 **매출 비중 최대 사업** 기준으로 귀속 (예: LG화학→화학, LG에너지솔루션→2차전지) |
| **커버리지 정합성** | Level 2는 증권사 애널리스트 리포트가 실제 발행되는 섹터 단위와 일치하도록 설계 → 리포트-산업 매핑이 자연스럽게 작동 |
| **골격 표준 준수** | Level 1은 WICS 10개 대분류를 따름. WICS는 네이버·다음 증권 및 에프앤가이드(컨센서스 데이터 제공사)가 채택한 사실상의 실무 표준 |
| **완전성 버킷** | 분류 곤란 종목은 "기타" 버킷으로 흡수해 누락 방지 |
| **테마는 뷰로 분리** | "AI", "로봇" 같은 테마는 분류가 아닌 **태그(다대다)**로 별도 관리 |

## 2. 검토 결론: 애널리스트 분류와의 관계 (v1.1 신설)

**검증 결과 요약** (WICS 방법론 원문 + 네이버/한경컨센서스 구조 확인, 2026-06-11):

1. **증권사 간 표준 애널리스트 분류는 존재하지 않음.** 리서치센터마다 커버리지 묶음이 다름 (화장품/의류 vs 유통/화장품 등). 한경컨센서스도 리포트를 기업/산업/시장/경제/파생으로만 구분.
2. **집계 레이어의 사실상 표준은 WICS** (10 대분류 / 28 중분류 / 79 소분류). 네이버·다음 증권, 에프앤가이드 리포트 분류가 모두 WICS 기준.
3. **따라서 본 체계의 전략**: Level 1 = WICS 대분류 그대로, Level 2 = WICS 중·소분류를 애널리스트 커버리지 단위로 재구성. 리포트 원문의 업종 표기는 **별칭(alias) 테이블**로 Level 2에 정규화.

**WICS 대비 의도적 차이 (매핑 시 오버라이드 필요):**

| Level 2 | WICS상 위치 | 처리 규칙 |
|---|---|---|
| 2차전지 | 셀 → 전자와전기제품(전기제품), 소재 → 소재(화학) 로 분산 | 셀·소재·장비를 통합한 단일 산업으로 정의. WICS 자동 매핑 후 종목 단위 오버라이드 목록 적용 (LG에너지솔루션, 삼성SDI, 에코프로비엠, 포스코퓨처엠, 엘앤에프 등) |
| 반도체 대형/소부장 분리 | WICS는 "반도체와반도체장비" 단일 | 커버리지 밀도가 높아 분리 유지. 매핑은 시총 기준 자동 분기 |
| 방산·우주항공, 조선, 전력기기, 지주·복합 | WICS 자본재 중분류의 소분류들 (우주항공과국방/조선/전기장비/복합기업) | 소분류를 Level 2로 승격 — WICS 소분류와 1:1 대응이라 매핑 충돌 없음 |

### 기타 버킷 *(v1.1 명시)*

WICS상 존재하나 애널리스트 커버리지가 거의 없는 산업(교육서비스, 상업서비스와공급품, 종이·목재, 포장재 등)과 분류 곤란 종목은 소속 Level 1의 "기타" 버킷으로 귀속. 시그널 계산에서 제외하되 종목 조회는 가능하게 유지. 스팩은 분석 대상 제외.

## 4. 리포트 업종 표기 정규화 (alias 테이블) *(v1.1 신설)*

증권사 리포트의 업종 표기는 표준화되어 있지 않으므로, 크롤링 시 다음 정규화를 적용:

```
report_sector_alias(raw_label, level2_id)
예: "인터넷/게임" → 분리 규칙: 대상 종목 기준 재귀속
    "유통/화장품" → 종목 코드로 Level 2 판정 (리포트의 업종 표기보다 종목 귀속 우선)
    "전기전자"   → IT하드웨어·전자부품
```

**핵심 규칙: 산업 귀속의 1차 기준은 리포트에 적힌 업종명이 아니라 리포트가 다루는 종목코드.** 종목→Level 2 매핑이 확정되어 있으므로, 종목 리포트는 자동으로 정확히 귀속됨. 업종명 alias는 종목이 특정되지 않는 산업 리포트에만 사용.

## 5. 커버리지 밀도와 시그널 엔진의 관계

| 밀도 | 의미 | 시그널 처리 |
|---|---|---|
| ★★★ | 주간 리포트 다수 | 4주 윈도우 정상 작동 |
| ★★ | 주 1~수 건 | 윈도우 8주로 확장 |
| ★ | 월 수 건 이하 | 최소 리포트 수 미달 시 "데이터 부족" 표시, Breadth 보류 |

## 6. 매핑 운영 규칙

1. **초기 적재**: KRX 상장종목 전체를 WICS 기준 1차 자동 매핑 → Level 2 변환 테이블 적용 → 2차전지 등 오버라이드 목록 반영
2. **수동 보정 큐**: 신뢰도 낮은 종목(지주, 복합사업, 사업전환)은 보정 대기열에서 수동 확정
3. **대표기업 동적 선정**: 본 문서의 대표기업은 시드값. 운영 시 *시가총액 상위 + 리포트 발행 빈도 상위* 교집합으로 분기 자동 갱신
4. **신규 상장·변경 처리**: 신규 상장은 보정 큐 경유, 합병·분할 시 재분류 이벤트 기록
5. **테마 태그 레이어**: AI, 로봇, 원전, K-푸드 등은 `theme_tags` 다대다 테이블로 독립 관리

## 7. DB 스키마 시사점

```
sectors(level1_id, level1_name)                       -- 10행
industries(level2_id, level1_id, level2_name,         -- 31행 + 기타
           coverage_density, signal_window_weeks)
companies(ticker, name, level2_id, market,
          mapping_confidence, is_representative)
mapping_overrides(ticker, level2_id, reason)          -- 2차전지 등 (v1.1)
report_sector_alias(raw_label, level2_id)             -- 업종명 정규화 (v1.1)
industry_etfs(level2_id, etf_ticker, etf_name)
theme_tags(tag_id, tag_name)
company_themes(ticker, tag_id)
```

## 8. 부수 발견: 애널리스트 정확도 데이터 *(v1.1 신설)*

한경컨센서스가 애널리스트별 평가 정확도 스코어·랭킹을 제공 중. Phase 4로 계획했던 "주요 애널리스트 선정"을 자체 적중률 트래킹 축적 전에 이 데이터로 부트스트랩 가능 → 기획서 로드맵 갱신 시 반영 예정.
