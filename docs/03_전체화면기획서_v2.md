## 0. 정보 구조(IA)와 화면 흐름

### 0.1 화면 맵

```
                    ┌──────────────┐
   텔레그램 알림 ──▶│ S1 오늘의 브리핑│◀── 매일 아침 진입점
   (딥링크)         └──┬───┬───┬───┘
                       │   │   │
        ┌──────────────┘   │   └──────────────┐
        ▼                  ▼                  ▼
  ┌───────────┐     ┌───────────┐      ┌───────────┐
  │ S2 워치 피드│◀───▶│ S3 산업 상세│◀───▶│ S5 거시 현황│
  └─────┬─────┘     └─────┬─────┘      └───────────┘
        │                 │
        └────────┬────────┘
                 ▼
          ┌───────────┐
          │ S4 종목 상세│
          └───────────┘
```

- 모든 화면은 S1의 드릴다운. 아침 사용 시나리오: 텔레그램 모닝 브리핑 → S1 (3분) → 관심 산업만 S3 → 필요 시 S4/원문
- 하단 고정 탭바 5개: [오늘] [워치] [산업] [종목] [거시]
- 산업/종목 탭은 "최근 본 항목" 목록이 기본 화면, 검색으로 임의 진입 가능

### 0.2 공통 규칙

| 항목 | 규칙 |
|---|---|
| 데이터 기준 표시 | 모든 화면 상단에 "데이터 기준: MM-DD HH:MM" (마지막 배치 완료 시각) |
| 딥링크 | 모든 텔레그램 알림은 카드 앵커(`#card-{id}`)로 연결. 탭 한 번에 해당 위치 도달 |
| 렌더링 | 사전 생성 JSON + 클라이언트 JS(Plotly.js + 바닐라). 페이지 전환 없는 탭 구조 |
| 반응형 | 모바일: 카드 1열, 탭바 하단 고정 / 데스크톱: 2~3열 그리드, 탭바 상단 |
| 색 체계 | 상승/긍정 = 녹색, 하락/부정 = 적색, 중립/보류 = 회색. 시그널 스코어는 0~100 |

## 1. 공통 컴포넌트

### C1. 리포트 카드 (S1·S2·S3·S4에서 재사용)

서비스 전체에서 리포트는 단일 카드 규격으로 표시한다.

```
┌─────────────────────────────────────────┐
│ {날짜 시각} · {산업} · {종목명}           │  헤더 (종목 미특정 산업리포트면 산업만)
│ {애널리스트} · {증권사}  [★워치]         │  ★는 산업별 Top3일 때만
│ 목표가 {직전} → {신규} ({±%})  {투자의견} │  변경 없으면 "유지", ±10%↑는 테두리 강조
│ ─ 요약 (있을 때만) ─────────────         │
│ 야마: "{1줄}"                            │
│ · 근거1  · 근거2  · 근거3                │
│ 제목: "{원문 제목}" (요약 없을 때만 표시) │
│ 원문: [한경/네이버] [{증권사} 리서치]     │  ← 항상 표시되는 핵심 영역
│ [산업 보기] [종목 보기] [요약 요청*]      │  *요약 없는 경우만
└─────────────────────────────────────────┘
```

**원문 링크 2종 (모든 카드 필수)**
- `source_url` 집계 사이트 링크(한경컨센서스/네이버 리서치 상세) — 항상 존재
- `broker_url` 증권사 리서치센터 링크 — 크롤링으로 확보 시 표시, 미확보 시 버튼 숨김
- 사용 흐름 전제: 사용자는 증권사 계정 보유, broker_url로 이동해 직접 로그인·다운로드. 서비스는 원문을 보관하지 않아도 가치 유지
- broker_url 미확보 증권사는 Phase 1에서 확보율 측정 후 "증권사명 검색 안내" 폴백 검토

**상태 변형**
- 요약 있음: 야마+근거 펼침 (기본 접힘/펼침은 워치 여부 따름 — 워치는 펼침)
- 요약 없음: 제목 표시 + [요약 요청] 버튼 (GitHub Issue 트리거 → 1~2분 내 재배포)
- 산업 리포트(종목 미특정): 목표가 영역 없음, 종목 버튼 없음

### C2. 산업 칩

산업명 + 시그널 스코어 + 변화 화살표를 묶은 최소 단위. 어디서든 탭하면 S3.
`[조선 87 ▲]`  `[화장품 55 ▼]`  `[의류 — (데이터 부족)]`

### C3. 지표 게이지 3종

Breadth(-1~+1), Magnitude(%), Upside Gap(%)을 각각 미니 바/숫자로. S1 카드와 S3 헤더에서 동일 규격 사용. 보정 감점이 적용된 경우 게이지 옆에 감점 배지(`후행성 -12`).

## 2. S1 — 오늘의 브리핑 (메인)

**목적**: 아침 3분 안에 "오늘 뭘 볼지" 결정 완료. 정보 밀도보다 판단 속도 우선.

### 레이아웃

```
┌────────────────────────────────────────────┐
│ 2026-06-12 (금) · 데이터 기준 06:30         │
│ 거시 신호등: 금리─ 환율▲ 수출▲ CPI─  → S5  │ ①
├────────────────────────────────────────────┤
│ ◆ 오늘의 주목 산업                          │
│ ┌─ 1위 카드 (확장형) ──────────────────┐   │
│ │ 1위 조선  스코어 87  [신규진입]       │   │
│ │ Breadth +0.78 │ Mag +9.4% │ Gap 18%   │   │ ②
│ │ "4주간 12건 중 11건 목표가 상향.      │   │
│ │  신조선가 상승이 컨센서스 반영 시작"  │   │
│ │ 신규 리포트 3 · 워치 1  → S3          │   │
│ └───────────────────────────────────────┘   │
│ [2위 전력기기 81] [3위 방산 76] (축약형)    │
├────────────────────────────────────────────┤
│ ◆ 섹터 맵 — 산점도                          │
│   y=Magnitude, x=Breadth, 버블=Gap          │ ③
│   회색 버블 = 데이터 부족 산업               │
├────────────────────────────────────────────┤
│ ◆ 시그널 변화 (전일 대비)                   │
│  ▲ 진입: 조선 (64→87)                       │ ④
│  ▼ 이탈: 화장품 (72→55 · 후행성 감점)       │
├────────────────────────────────────────────┤
│ ◆ 워치 애널리스트 최신 5건 (C1 축약) → S2   │ ⑤
└────────────────────────────────────────────┘
```

### 기능 상세

| ID | 기능 | 설명 | 데이터/갱신 |
|---|---|---|---|
| F1-1 | 거시 신호등 | 기준금리·환율·수출YoY·CPI 방향(▲▼─) 한 줄 압축. 탭 → S5 | ECOS/KOSIS, 일 1회 |
| F1-2 | 주목 산업 Top 3 | 1위는 확장 카드(3대 지표 + LLM 근거 1줄 + 리포트 수), 2~3위는 축약 칩. LLM 근거 1줄은 해당 산업의 실제 리포트 건수·제목을 인용한 한 문장 (생성 실패 시 룰 기반 템플릿 문장으로 폴백) | 시그널 엔진 + LLM, 일 1회 |
| F1-3 | 섹터 맵 | 전 산업 산점도. 데이터 부족 산업은 회색 + 툴팁. 버블 탭 → S3 | 시그널 엔진, 일 1회 |
| F1-4 | 시그널 변화 | 후보군 진입/이탈과 사유(지표·보정 명시). 시그널 설명가능성의 핵심 장치 | 시그널 엔진, 일 1회 |
| F1-5 | 워치 미리보기 | C1 카드 축약형 5건. "더보기" → S2 | 크롤러, 일 2회 |

### 레이아웃

```

├────────────────────────────────────────────┤
│ C1 카드 — 요약 있음 (펼침)                  │
│ C1 카드 — 요약 없음 (출시 알림형)           │
│ C1 카드 (접힘, 비워치) …                    │
│ … 시간 역순, 스크롤 로드                    │
└────────────────────────────────────────────┘
```

| ID | 기능 | 설명 | 데이터/갱신 |

## 5. S4 — 종목 상세

**목적**: 투자 로직 4단계 — 글로벌 경쟁력, 이익률 증가 속도 등 긍정 요소 점검.

### 레이아웃

```
┌────────────────────────────────────────────┐
│ 한화오션 (042660) · 조선 → S3               │
│ 현재가 41,200 ▲2.1% · 컨센서스 Gap +14%     │ ①
├────────────────────────────────────────────┤
│ ◆ 주가 + 목표가 밴드 (1년)                  │
│   ── 주가  ▒▒ high~low 밴드  ─ 평균        │ ②
├────────────────────────────────────────────┤
│ ◆ 수급 — 외국인·기관 누적 순매수 (60일)     │
│   외국인 지분율 추이 병기                   │ ③
├────────────────────────────────────────────┤
│ ◆ 펀더멘털 (분기)                           │
│  영업이익률 4.2 → 5.1 → 6.8 [2분기 연속▲]   │ ④
│  매출 YoY · 영업이익 컨센서스 변화          │
├────────────────────────────────────────────┤
│ ◆ 정성 요소 (LLM 추출 · 출처 필수)          │
│  · "LNG선 점유율 ○○%" — 6/12 현대차증권    │ ⑤
│  · "수주잔고 3.2년치" — 6/10 △△증권        │
├────────────────────────────────────────────┤
│ ◆ 이 종목 리포트 (최근 10건, C1 카드)       │ ⑥
└────────────────────────────────────────────┘
```

### 기능 상세

| ID | 기능 | 설명 |
|---|---|---|
| F4-1 | 목표가 밴드 차트 | 1년 주가 + 컨센서스 최고/평균/최저 밴드. 목표가의 주가 선행 여부 육안 확인 (보정 로직의 종목판) |
| F4-2 | 수급 차트 | 외국인·기관 60일 누적 순매수 + 외국인 지분율 |
| F4-3 | 펀더멘털 추이 | DART 분기 데이터. 보조 시그널 충족 시 배지("2분기 연속 개선") |
| F4-4 | 정성 요소 | 글로벌 점유율·수주잔고 등 비정형 정보를 LLM이 리포트에서 추출. **출처(날짜·증권사) 필수, 원문 인용 없는 생성 금지**. 추출 실패 시 섹션 숨김 |
| F4-5 | 종목 리포트 목록 | C1 카드 재사용 (원문 링크 2종 포함) |

## 6. S5 — 거시 현황

**목적**: 투자 로직 1단계의 경량 확인 — "현재 국면이 산업 시그널을 지지하는가."

### 레이아웃

```
┌────────────────────────────────────────────┐
│ ◆ 핵심 지표 카드 (4)                        │
│  기준금리 X.XX% ─   (M/D 금통위 동결)       │
│  원/달러 1,XXX ▲    (1개월 +1.8%)           │ ①
│  수출 YoY +X.X% ▲   (반도체 +XX%)           │
│  CPI YoY X.X% ─                             │
│  각 카드: 현재값 + 방향 + 12개월 스파크라인 │
├────────────────────────────────────────────┤
│ ◆ 산업 연결 메모 (룰 기반, LLM 아님)        │
│  "수출 개선이 조선·반도체 Breadth 상승과    │ ②
│   동행 — 시그널 신뢰도 보강"                │
└────────────────────────────────────────────┘
```

| ID | 기능 | 설명 |
|---|---|---|
| F5-1 | 지표 카드 | ECOS/KOSIS 4지표 + 12개월 스파크라인 + 발표 이벤트 날짜 |
| F5-2 | 산업 연결 메모 | 거시 지표와 산업 시그널의 동행/괴리를 룰 템플릿 문장으로 (비용 0) |

| 첫 가동기(데이터 < 윈도우) | S1에 "데이터 축적 중 D+{n}/28 — 시그널은 {date}부터 신뢰 가능" 안내 |
