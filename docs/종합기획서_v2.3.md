## 1. 서비스 개요와 운영 전제

국내 상장 주식 시장을 **산업(섹터) 단위로 상시 모니터링**하여 *"지금 어떤 산업의 어떤 기업을, 왜 봐야 하는지"*를 제시하는 **1인용 재무 분석 서비스**. 거시 → 산업 → 기업으로 좁혀가는 탑다운 투자 로직을 데이터 파이프라인과 룰 기반 시그널 엔진으로 시스템화한다.

> **서비스 목적 (스코프 고정, v2.2)**: 이 서비스는 *일간 이슈 트래커가 아니다.* 거시적 **산업 사이클 전환(다운턴→업턴)을 남보다 먼저 포착**하고, 포착된 산업의 **대표기업·ETF·애널리스트 리포트를 묶어** 사용자가 **본격 분석에 들어가기 직전 단계를 점검**해 주는 도구다. 따라서 화면·알림·신호는 일간 변화보다 **주간 트렌드와 사이클 국면**을 우선한다. (백엔드 데이터 수집은 일간 유지 — 일간 데이터가 쌓여야 주간 추세·전환을 계산할 수 있으므로.)

| 전제 | 내용 |
|---|---|
| 사용자 | 1인 (본인). 개인 투자 분석 도구 |
| 비용 | 0원 수렴 최우선. 상시 서버 없음(정적 아키텍처), LLM은 무료 티어 |
| 산출물 책임 | 시그널은 참고 자료. 최종 의사결정·책임은 사용자 본인 |

## 3. 발굴 로직 — 신호 엔진 (★ v2.1 핵심)

### 3.1 기본 사상

1순위 신호는 **"섹터 내 애널리스트 전망이 동시에, 큰 폭으로 상향되는 현상"**. 개별 상향은 종목 이벤트일 수 있으나 섹터 단위 동시 상향은 산업 펀더멘털 변화의 증거다. **절대 수준이 아닌 변화(revision)**를 추적한다. 어떤 산업/종목을 볼지는 **룰 엔진이 결정**하고, LLM은 그 결과의 설명·요약만 한다.

### 3.2 산업 선별 — 2레이어 (관절②)

**확인 레이어 — 목표가 모멘텀 3지표** (윈도우 4~8주)

| 지표 | 정의 | 해석 |
|---|---|---|
| 상향 폭 Magnitude | 섹터 내 목표주가 변경률 평균 | 얼마나 크게 올리는가 |
| 상향 확산도 Breadth | (상향 − 하향) / 전체 리포트 수 | 다같이 오르는가 (쏠림 배제) |
| 상승 여력 Upside Gap | 섹터 평균 (목표가 − 현재가) / 현재가 | 진입 매력 잔존 |

**선행 레이어 — 더 일찍 도는 신호** (v2.1 신설)

| 지표 | 정의 | 근거 |
|---|---|---|
| EPS 리비전 B/M | 목표가 대신 **EPS 추정치**에 Breadth·Magnitude 동일 적용 | 이익 추정치 리비전이 목표가 리비전을 **선행** |
| 리비전 가속도 | Δ(리비전율) = 직전 윈도우 대비 변화 (2차 미분) | *리비전의 가속*이 변곡점 포착 — "이제 막 도는" 섹터 |
| 최초 전환 탐지 | Breadth가 음→양으로 **가장 최근 교차**한 섹터 플래그 | "확인된" 섹터가 아니라 *가장 먼저 도는* 섹터 = 발굴 우선 |

**신호 우선순위 (대시보드 정렬)**

```
1순위: 최초 전환 + EPS 리비전 가속도(+)        → "지금 막 발굴할 섹터"
2순위: 목표가 모멘텀 확인 (Breadth ≥ +0.6)     → "이미 확인된 섹터" (확신용)
```

선행만 켜짐 = 관찰 후보 / 둘 다 켜짐 = 신뢰도 최상 / 확인만 켜짐 = 추격 주의.

### 3.3 보정 장치 (필수, 양 레이어 공통)

1. **후행성 보정**: 주가 상승률 대비 전망 상향률 비교 — 전망이 주가를 앞서면 선행(가점), 뒤따르면 추격(감점)
2. **희소 섹터 보정**: 윈도우 내 최소 5건 미달 섹터는 시그널 보류 + "데이터 부족" 표시 (커버리지 밀도 등급 연동)

### 3.4 종목 선별 — 섹터 내 펀더멘털 모멘텀 (관절③, v2.1 신설)

섹터 신호 발동 후, 그 안의 종목을 다음 점수로 랭킹한다.

```
종목_점수 = z(EPS 추정치 리비전 %)      × 0.5     # 이익 전망이 오르는가
          + z(영업이익률 추세, 2분기 Δ)  × 0.5     # 실제 수익성이 개선되는가
보조: 동점 시 Upside Gap으로 타이브레이크 (1차 점수 미포함)
```

산업 귀속은 종목코드 기준(원칙 4). 종목 점수는 **이미 Level2에 귀속된 종목들 사이에서만** 계산한다.

### 3.4-b 사이클 국면 분류 (v2.2 — 1급 개념)

스코어(0~100)는 "얼마나 강한가"만 말해줄 뿐, 사용자의 핵심 질문 *"이 산업이 사이클 어디에 있나 — 지금 도는가"*에 답하지 못한다. 그래서 각 산업을 **5개 사이클 국면**으로 분류해 헤드라인에 노출한다. (별도 데이터 불필요 — 기존 Breadth·가속도·Magnitude·Gap을 국면으로 번역.)

| 국면 | 신호 조건 | 의미 | 색 |
|---|---|---|---|
| **침체** (다운턴) | Breadth < 0, 가속도 − | 전망 하향 지속 | 회/적 |
| **바닥** | Breadth < 0, **가속도 +** | 하향세 둔화, 개선 시작 | 청 |
| **전환** (업턴 초입) | Breadth **음→양 교차** (최초 전환) | **다운턴→업턴 — 핵심 캐치 존** | 녹(강조) |
| **확장** | Breadth > 0 지속, Magnitude 견조 | 상향 본격화 | 녹 |
| **둔화** | Breadth > 0이나 Magnitude 약화 / Gap 축소 | 과열·후행, 진입 여력 소진 | 주황 |

**운영 원칙**: 대시보드는 "전환" 국면 산업을 최상단에 고정 노출(발굴 우선순위). "확장"은 이미 가는 중(추격 주의), "바닥"은 관찰 후보. 국면 *전이*(바닥→전환, 확장→둔화)를 주간 단위로 추적해 알림한다 — 이것이 일간 이벤트 알림을 대체하는 핵심 신호다.

### 3.5 거시 = 참고 화면 (관절①, 미연결 확정)

거시 지표를 산업 스코어에 **능동 반영하지 않는다.** 거시→산업 민감도 맵은 비선형·국면의존적이라 룰 고정 시 오신호 위험이 크고, 1인 사용자가 스스로 잘 판단하는 영역이다. 화면(S5)의 "산업 연결 메모"는 **동행/괴리를 서술만** 할 뿐 스코어를 바꾸지 않는다. → `industry_macro_sensitivity` 류 테이블/로직은 만들지 않는다.

### 3.6 보조 시그널 (2차 필터)

- 산업: 업종지수 골든크로스, 업종 ETF 거래대금 증가, 수출 YoY 플러스 전환, 외국인 업종 순매수 전환
- 기업: 영업이익률 2분기 연속 개선, 컨센서스 상향, 외국인 지분율 증가, 글로벌 점유율(리포트 인용 LLM 보조)

## 5. 애널리스트 트래킹

### 5.1 산업별 Top 3 — 2단계 운영

**Phase A 부트스트랩 (~6개월)**: 어워드 시드(매경·한경비즈니스·FnGuide 산업별 수상자) + 한경 스코어 수동 참조로 산업당 3명. 검증 예시 — 자동차 장문수(현대차증권), 인터넷·SW 임희석(미래에셋증권). 시드 없는 산업은 발행 빈도 상위 3명.

**Phase B 자동 선정 (6개월~)**: 자체 적중률 엔진 가중 스코어 상위 3명, 분기 자동 교체.

### 5.2 자체 적중률 산출

```
적중률(r,h) = 100% − |TP − P(h)| / TP        # h = 6·12개월 주가
방향 점수    = sign(TP−P0)==sign(P(h)−P0) ? 1:0
가중 스코어  = z(정확도)×0.5 + z(방향)×0.3 + z(리포트수)×0.2
```

보완: 6/12개월 이중 측정, 방향 적중 별도 반영(보수적 목표가 왜곡 방지), 산업별 5건 미만 제외. 동명이인은 (이름, 증권사) 복합키 + predecessor_id로 식별.

### 5.3 리포트 수집 3-Tier

```
[감지] 네이버 리서치 폴링 (18:00 본 + 08:30 보조) → 신규 행 + 상세(애널리스트·목표가·의견)
[Tier1] PDF 공개분 → 텍스트 추출 → LLM 요약(야마1+근거3+목표가 맥락), 원문 로컬 보관
[Tier2] PDF 비공개 → 메타데이터 카드(제목·목표가 변경·링크) + "원문 미공개" 배지

```

**출시 감지·알림은 전체 리포트 100% 보장**(메타데이터 상시 공개), 요약은 공개분만.

### 8.1 정보 구조 (IA) — v2.3 재편: 깔때기 척추

탭을 ① 줌 레벨 축(거시→산업→종목, MECE 깔때기) ② 요약(진입·큐레이션) ③ 리포트(근거 피드)로 정리한다. "워치"는 탭이 아니라 리포트 탭의 필터로 강등.

```
                 ┌──────────┐
                 │   요약    │  진입·큐레이션 (이번 주 전환·주목만)
                 └────┬─────┘
   ┌──────────────────┼──────────────────┐   ← 탑다운 깔때기 (줌 레벨, MECE)
   ▼                  ▼                  ▼
┌──────┐  drill   ┌──────┐  drill   ┌──────┐
│ 거시 │ ───────▶ │ 산업 │ ───────▶ │ 종목 │
└──────┘          └───┬──┘          └───┬──┘
                      └───────┬─────────┘
                              ▼
                        ┌──────────┐
                        │  리포트   │  근거 피드 (애널리스트, 워치=필터)
                        └──────────┘
```

**하단 고정 탭바 5개: [요약][거시][산업][종목][리포트].** 산업·종목 탭은 **랜딩(목록)+상세(드릴다운)** 2단:

| 탭 | 랜딩 | 상세 |
|---|---|---|
| 요약 | 거시 카드·이번 주 국면 전환·전환 산업 hero·트렌드 순위 미리보기·최신 리포트 | — (각 항목이 해당 탭으로 딥링크) |
| 거시 | 4지표 카드 + **지표별 시계열 그래프(Y축 금액)** + 경제 캘린더 + 산업 연결 메모 | — |
| 산업 | **산업 트렌드 순위 목록**(국면·종합의견·목표가Δ) | 산업 상세(사이클 국면·분석 시작 패키지·타임라인·종목 랭킹) |
| 종목 | **산업별 대표 종목 picker**(산업 칩 → 종목 리스트) | 종목 상세(주가/목표가 밴드·**기간 토글 10/5/3/1Y·3M**·수급·펀더멘털) |
| 리포트 | 애널리스트 리포트 피드(워치=필터) | C1 카드 펼침 |

연결성: 요약→각 탭 딥링크, 거시→산업→종목 drill, 산업 상세↔종목 상세 상호 링크, 리포트→산업/종목. 모든 화면 상단 "데이터 기준: MM-DD HH:MM". 모바일 우선(1열). 색: 상승=녹/하락=적/보류=회, 스코어 0~100.

### 8.2 공통 컴포넌트

## 11. 핵심 원칙 7 (CLAUDE.md, 절대 위반 금지)

1. **제로-서버·최소비용**: 상시 서버 금지, Actions 일 배치 + SQLite 커밋 + 정적 HTML. 유료 전 무료 대안 먼저
2. **LLM은 설명만**: 산업/종목 선정은 룰 엔진(산업=확인+선행 2레이어, 종목=펀더멘털 모멘텀, 거시 미연결). LLM은 요약·브리핑만, Gemini 기본·인터페이스 분리
3. **출처 명기 필수**: 정성 정보는 원문 날짜·증권사 표시. 출처 없는 생성 금지
4. **산업 귀속 = 종목코드 기준** (리포트 업종 표기 아님)
5. **MECE**: 모든 종목 정확히 1개 Level2, 곤란 시 "기타"
6. **저작권**: PDF·요약은 개인용 로컬 보관, 재배포 금지
7. **리포트 카드는 원문 링크 2종** (source_url + broker_url)
