# 프로젝트: 국내 주식 산업 모니터링 서비스

## 개요
1인 사용 전제의 국내 상장주식 산업(섹터) 모니터링 서비스. 거시→산업→기업 탑다운 투자 로직을
시그널 엔진으로 시스템화하여 "지금 어떤 산업의 어떤 기업을, 왜 봐야 하는지"를 매일 자동 분석한다.

상세 기획은 docs/ 디렉토리 참조:
- docs/00_통합기획서_v2.md — 전체 개요, 아키텍처, 로드맵 (가장 먼저 읽을 것)
- docs/01_산업분류체계_v1.1.md — Level 1/2 산업 분류, 종목 매핑 규칙
- docs/02_애널리스트_트래킹_설계_v1.md — 애널리스트 Top3 선정, 리포트 수집
- docs/03_전체화면기획서_v2.md — 전체 화면 구성·공통 컴포넌트·기능 상세

## 핵심 원칙 (절대 위반 금지)

1. **제로-서버, 최소비용**: 상시 구동 서버 금지. 모든 연산은 GitHub Actions의 일 배치(cron)로
   수행하고 SQLite 파일을 repo에 커밋한다. 대시보드는 정적 HTML/JSON으로 생성한다.
   유료 API/서비스를 추가하기 전에 항상 무료 대안을 먼저 검토하고 사용자에게 비용을 명시한다.

2. **LLM은 판단하지 않고 설명만 한다**: 어떤 산업/종목을 주목할지는 룰 기반 시그널 엔진
   (목표주가 모멘텀 3지표: Breadth, Magnitude, Upside Gap + 보정 로직)이 결정한다.
   LLM은 그 결과에 대한 요약·브리핑 생성에만 사용한다.
   기본 LLM은 Gemini API(Flash, 무료 티어)이며, LLM 호출부는 `summarize(text) -> dict` 형태로
   인터페이스를 분리해 모델 교체 비용을 최소화한다. 목표가·종목명 등 수치/고유명사 추출
   정확도가 낮을 경우에만 해당 항목을 Claude(Haiku, Batch)로 부분 교체한다.

3. **출처 명기 필수**: 정성적 정보(글로벌 점유율, 수주잔고 등)를 LLM으로 추출할 때는
   반드시 원문 리포트의 날짜·증권사를 함께 표시한다. 출처 없는 생성 정보는 표시하지 않는다.

4. **산업 귀속은 종목코드 기준**: 리포트의 업종 표기(alias)가 아니라 종목코드→Level2 매핑
   테이블이 1차 기준이다. (이유: 증권사별 업종 표기가 표준화되어 있지 않음)

5. **MECE**: 모든 상장종목은 정확히 1개 Level2 산업에 귀속. 분류 곤란 종목은 "기타" 버킷.

6. **저작권**: 리포트 PDF/요약은 개인 분석용으로만 로컬(repo 내) 보관. 재배포 금지.

7. **모든 리포트 카드는 원문 링크 2종을 포함한다**: source_url(한경/네이버 등 집계 사이트,
   항상 존재)과 broker_url(증권사 리서치센터, 확보 시에만). 사용자가 증권사 계정으로 직접
   원문을 받는 것을 전제로 하므로, 서비스가 PDF 원문을 확보/요약하지 못해도 카드 자체의
   가치(출시 알림 + 두 링크)는 유지되어야 한다. 화면 구현 시 docs/03의 "C1. 리포트 카드"
   공통 컴포넌트 규격을 그대로 따른다.

## 기술 스택

- 언어: Python (pykrx, requests, BeautifulSoup, sqlite3)
- 저장소: SQLite 단일 파일 (data/monitor.db, repo에 커밋)
- 스케줄러: GitHub Actions (.github/workflows/)
- 대시보드: 정적 HTML + Plotly.js (클라이언트 사이드 렌더, 사전 생성 JSON 사용)
- LLM: Gemini API (Flash, 무료 티어 기본) — 인터페이스 분리로 Claude 등과 교체 가능
- 알림: 텔레그램 봇 API
- 거시/시세: 한국은행 ECOS, KOSIS, DART OpenAPI, pykrx — 전부 무료

## 외부 프로젝트와의 관계

이 프로젝트는 'DearName'(작명 서비스)과 완전히 독립된 별도 프로젝트다.
코드, 데이터, 설정, 의존성을 공유하지 않는다. DearName 관련 코드/파일을 이 저장소에
절대 포함하지 않는다.

## 작업 시 주의사항

- 계획 문서(기획서, 설계 변경 제안 등)는 항상 .md 파일로 docs/에 작성한다 (PDF 금지).
- 새 데이터 소스를 추가하기 전, 무료 공식 API가 있는지 먼저 확인한다.
- Phase 순서를 따른다: P1 데이터 파이프라인 → P2 시그널+대시보드 → P3 LLM 통합 → P4 백테스트/자동화.
  현재 Phase: docs/00_통합기획서_v2.md의 8장 로드맵 참조.
